# [알고팜 1주차] BOJ 26215 눈 치우기 - 복잡하게 생각한 그리디를 정렬로 다시 풀기

이번 문제는 처음 볼 때보다 다시 정리할 때 훨씬 이해가 잘 됐다.  
처음에는 "눈이 많은 집끼리 한 번에 최대한 많이 없애면 되지 않을까?"라는 식으로 생각해서 코드를 꽤 복잡하게 짰는데, 막상 반례를 만나고 나니 핵심은 훨씬 단순한 그리디였다.  
매 분마다 **지금 눈이 가장 많이 쌓인 두 집을 골라 1씩 줄인다**. 이 한 문장으로 정리되는 문제였다.

문제 링크: https://www.acmicpc.net/problem/26215

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 512MB
- 분류 느낌: 그리디 / 구현 / 시뮬레이션

## 문제 설명

겨울 숲의 여러 집 앞에 눈이 쌓여 있고, 눈이 오지 않는 동안 모든 집 앞의 눈을 치워야 한다.

- 1분에 두 집을 골라 각각 1만큼 치울 수 있다.
- 또는 집 하나만 골라 1만큼 치울 수도 있다.
- 모든 집 앞의 눈을 다 치우는 최소 시간을 구해야 한다.
- 다만 24시간, 즉 1440분을 넘기면 `-1`을 출력해야 한다.

문제를 읽고 나면 "한 번에 가장 많이 치우는 방법"을 찾고 싶어지는데, 실제로는 매 순간 가장 큰 두 집을 고르는 단순한 그리디가 핵심이다.

## 입력 형식

- 첫째 줄에 집의 수 `N`이 주어진다.
- 둘째 줄에 각 집 앞의 눈의 양 `a_i`가 공백으로 주어진다.
- 문제 조건은 `1 <= N <= 100`, `1 <= a_i <= 2000`이다.

## 출력 형식

- 모든 집의 눈을 치우는 최소 시간을 출력한다.
- 필요한 시간이 `1440`분을 넘으면 `-1`을 출력한다.

## 예제 입력 1

```text
3
1 2 3
```

## 예제 출력 1

```text
3
```

## 예제 입력 2

```text
3
1 2 5
```

## 예제 출력 2

```text
5
```

## 예제 입력 3

```text
1
1441
```

## 예제 출력 3

```text
-1
```

## 접근 방법

이 문제에서 한 분에 할 수 있는 최선의 행동은 결국 두 집의 눈을 동시에 1씩 줄이는 것이다.  
그렇다면 어떤 두 집을 골라야 할까?

내가 다시 정리하면서 잡은 기준은 이거였다.

1. 가장 눈이 많은 집을 계속 방치하면 마지막에 그 집 하나 때문에 시간이 길어진다.
2. 그래서 매 분마다 현재 눈이 가장 많은 두 집을 먼저 줄이는 게 자연스럽다.
3. 이 과정을 눈이 모두 없어질 때까지 반복하면 된다.

즉, 이 문제는 "한 번에 많이 빼는" 문제가 아니라, **매 분마다 다시 정렬해서 가장 큰 두 집을 고르는 문제**로 보면 훨씬 단순해진다.

## 내가 처음 작성했던 코드

처음에는 아래처럼 짰다.

```python
import sys

input = sys.stdin.readline

n = int(input())
list1 = list(map(int, input().split()))

cnt = 0

if n == 1:
    cnt += list1[0]
else:
    while sum(list1) != 0:
        list1.sort(reverse=True)
        if list1[0] != 0 and list1[1] != 0:
            cnt += list1[1]
            list1[0] -= list1[1]
            list1[1] = 0
        else:
            cnt += list1[0]
            list1[0] = 0

if cnt > 1440:
    print(-1)
else:
    print(cnt)
```

## 이 코드에서 내가 하려던 것

이 코드는 분명히 방향은 그리디였다.

- 매번 내림차순 정렬을 해서 눈이 가장 많은 집부터 본다.
- 첫 번째 집과 두 번째 집을 같이 처리하는 게 이득이라고 생각한다.
- 그래서 두 번째 집의 눈이 모두 없어질 때까지 첫 번째 집과 한꺼번에 묶어서 없애려고 한다.

처음 떠올리기에는 꽤 그럴듯한 방식이다.  
나도 처음에는 "가장 큰 두 집을 묶어서 최대한 빨리 없애면 되겠지"라고 생각했다.

## 약했던 점과 반례

문제는 이 코드가 **한 분씩 다시 판단하지 않는다**는 점이다.

이 문제의 핵심은:

- 1분 치운다.
- 그 다음 현재 상태를 다시 본다.
- 다시 가장 큰 두 집을 고른다.

이 흐름이어야 하는데, 처음 코드에서는 `list1[1]`만큼을 한 번에 빼 버린다.  
그러면 중간에 짝을 바꿔야 하는 경우를 놓치게 된다.

대표적인 반례가 바로 이 경우다.

```text
3
2 2 2
```

처음 코드 방식으로 계산하면 4분처럼 보이지만, 실제 최소 시간은 3분이다.

- 1분차: 두 집을 골라 `2 2 2 -> 1 1 2`
- 2분차: 다시 가장 큰 두 집을 골라 `1 1 2 -> 1 0 1`
- 3분차: 남은 두 집을 골라 `1 0 1 -> 0 0 0`

즉, "한 번 정한 두 집을 끝까지 같이 치운다"가 아니라 **매 분마다 현재 최댓값 두 개를 다시 골라야 한다**.

## 개선 풀이 1: 정렬로 푸는 가장 직관적인 그리디

지금 내가 최종적으로 정리한 코드는 이 버전이다.

```python
import sys

input = sys.stdin.readline

n = int(input())
snow = list(map(int, input().split()))

time = 0

while True:
    snow.sort(reverse=True)

    if snow[0] == 0:
        print(time)
        break

    snow[0] -= 1

    if n > 1 and snow[1] > 0:
        snow[1] -= 1

    time += 1

    if time > 1440:
        print(-1)
        break
```

### 왜 이 풀이가 더 이해하기 쉬웠는가

이 코드는 한 바퀴가 정확히 1분을 의미한다.

- 매 분마다 내림차순 정렬한다.
- 눈이 가장 많은 집 하나를 1 줄인다.
- 두 번째 집도 남아 있으면 1 줄인다.
- 시간을 1 늘린다.

문제의 조건을 거의 그대로 코드로 옮긴 셈이라서, 내가 다시 읽어도 흐름이 바로 보였다.  
입력 크기도 크지 않아서 `sort(reverse=True)`를 매번 써도 충분히 통과 가능하다.

## 개선 풀이 2: 나중에 알게 된 더 짧은 수식 풀이

그리디로 푸는 게 문제 이해에는 가장 좋았지만, 나중에 다시 보니 이 문제는 수식으로도 정리할 수 있다.

```python
import sys

input = sys.stdin.readline

n = int(input())
snow = list(map(int, input().split()))

total = sum(snow)
largest = max(snow)
answer = max(largest, (total + 1) // 2)

print(-1 if answer > 1440 else answer)
```

### 왜 이 수식이 맞는가

이유는 하한을 두 개로 보면 된다.

- 어떤 한 집도 1분에 최대 1만큼만 줄일 수 있으니 최소 시간은 적어도 `max(snow)` 이상이다.
- 전체로는 1분에 최대 2만큼만 줄일 수 있으니 최소 시간은 적어도 `ceil(sum(snow) / 2)` 이상이다.

결국 최소 시간은 두 값 중 큰 값보다 작아질 수 없고, 실제로 그 시간 안에 그리디하게 치우는 것도 가능하다.  
문제를 한 번 그리디로 이해한 다음에 보면 이 식이 왜 나오는지도 훨씬 납득이 된다.

## 시간 복잡도

- 처음 작성한 코드: `O(K * n log n)`
- 개선 풀이 1: `O(T * n log n)`
- 개선 풀이 2: `O(n)`

이유도 같이 정리해 보면:

- 처음 작성한 코드는 반복할 때마다 정렬을 하므로 한 번 반복에 `O(n log n)`이 든다. 다만 한 번에 둘째 집의 눈을 통째로 없애는 식이라 반복 횟수를 직관적으로 세기 어려워서 `K`번 반복한다고 두는 편이 자연스럽다.
- 개선 풀이 1은 매 분마다 정렬을 한 번 하고, 그 정렬이 `O(n log n)`이다. 이 과정을 실제 걸린 시간 `T`분만큼 반복하므로 `O(T * n log n)`이다.
- 개선 풀이 2는 `sum`과 `max`만 한 번씩 구하면 끝난다. 둘 다 리스트를 한 번 훑는 연산이라 전체가 `O(n)`이다.

이 문제는 `N <= 100`이고, 어차피 `1440`분이 넘으면 바로 `-1`이어서 정렬 그리디도 충분히 안전하다.

## 이번 문제에서 배운 점

- 그리디 문제는 "한 번에 많이 처리하는 것"보다 "매 순간 다시 최선 선택을 하는 것"이 더 중요할 때가 많다.
- 시뮬레이션이 길어 보여도 문제의 행동 단위를 그대로 옮기면 오히려 코드가 단순해질 수 있다.
- 정답 코드를 억지로 복잡하게 만들기보다, 내가 다시 읽었을 때 설명 가능한 코드가 훨씬 좋은 풀이인 것 같다.

## 마무리

이번 문제는 처음엔 꽤 어렵게 느껴졌는데, 다시 보니 오히려 그리디의 감각을 되짚기 좋은 문제였다.  
특히 내가 처음 짠 코드처럼 "두 집을 한 번에 많이 빼는 방식"은 얼핏 맞아 보여도, 실제로는 매 분마다 최댓값 두 개를 다시 골라야 한다는 점을 놓치기 쉽다.

결국 이번 문제에서 제일 크게 남은 포인트는 이거였다.

**그리디는 한 번 정하고 밀어붙이는 게 아니라, 매 순간 다시 선택하는 방식이다.**
