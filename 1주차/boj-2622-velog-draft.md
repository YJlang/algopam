알고팜 1주차에서 풀었던 **BOJ 2622 - 삼각형만들기**를 정리해보려고 한다.  
처음에는 가능한 변 길이를 직접 세는 방식으로 접근했고, 다시 정리해보니 이 문제는 **브루트포스에서 출발해서 수학적으로 줄여가는 과정**이 꽤 재미있는 문제였다.

문제 링크: https://www.acmicpc.net/problem/2622

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현 / 브루트포스 / 수학

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 길이가 같은 성냥개비가 `N`개 주어진다.
- 이 성냥개비를 전부 사용해서 삼각형 한 개를 만들어야 한다.
- 한 변은 여러 성냥개비를 일자로 이어서 만들 수 있지만, 꺾거나 자를 수는 없다.
- 합동인 삼각형은 같은 삼각형으로 본다.
- 만들 수 있는 서로 다른 삼각형의 개수를 구하면 된다.

핵심은 결국 **정수 길이 세 변 `a, b, c`를 만들되**

- `a + b + c = N`
- `a <= b <= c`
- `a + b > c`

를 만족하는 경우의 수를 세는 문제라고 볼 수 있다.

## 입력 형식

- 첫째 줄에 성냥개비의 개수 `N`이 주어진다.
- `1 <= N <= 50000`

## 출력 형식

- 만들 수 있는 서로 다른 삼각형의 개수를 출력한다.

## 예제 입력 1

```text
9
```

## 예제 출력 1

```text
3
```

## 접근 아이디어

내가 처음 생각한 방향은 아래와 같았다.

1. 세 변의 길이를 `i`, `ii`, `iii`라고 두고 직접 만들어본다.
2. 중복을 막기 위해 `i <= ii <= iii` 형태만 센다.
3. 삼각형 조건 `i + ii > iii`를 만족하면 개수를 증가시킨다.

즉, 가능한 삼각형을 하나씩 직접 세는 브루트포스 접근이었다.

## 내가 처음 작성한 코드

```python
import sys
input = sys.stdin.readline

N = int(input())
cnt = 0

for i in range(1, N+1):
    for ii in range(i, N+1):
        iii = N - (i+ii)
        
        if iii >= i + ii:
            continue
        else:
            if ii > iii:
                break
            cnt += 1

print(cnt)
```

## 이 코드에서 내가 하려고 했던 것

이 코드는 사실상 다음 역할을 하고 있다.

- `i`를 가장 짧은 변
- `ii`를 그다음 변
- `iii = N - (i + ii)`를 마지막 변

으로 놓고, 세 변의 합이 항상 `N`이 되도록 맞춘다.

그다음

- `iii >= i + ii`이면 삼각형이 아니므로 건너뛴다.
- `ii > iii`이면 정렬 조건 `i <= ii <= iii`가 깨졌으므로 더 볼 필요 없이 `break`한다.
- 그 외에는 유효한 삼각형이므로 `cnt`를 증가시킨다.

즉, **직접 세 보되 중복은 정렬 조건으로 제거한 코드**라고 정리할 수 있다.

## 아쉬웠던 점

아이디어 자체는 맞지만, 이 방식은 제한을 생각하면 꽤 무겁다.

- 바깥 루프가 거의 `N`번 돈다.
- 안쪽 루프도 경우에 따라 많이 반복된다.
- 결국 Python 기준으로는 대략 `O(N^2)`에 가까운 탐색이 된다.

이 문제는 `N`이 최대 `50000`이기 때문에,  
정수 조합을 전부 직접 확인하는 방식은 성능 면에서 부담이 크다.

또 하나 아쉬운 점은 변수 이름이다.

- `i`, `ii`, `iii`는 빠르게 짤 때는 편하지만
- 블로그로 설명하거나 나중에 다시 볼 때는 의미가 바로 들어오지 않는다.

## 개선 풀이 1: 가능한 범위를 한 번에 세기

이 문제의 핵심은 세 변을 전부 직접 만들 필요가 없다는 점이다.  
가장 짧은 변 `a`를 하나 정하면, 두 번째 변 `b`가 될 수 있는 범위를 바로 구할 수 있다.

```python
import sys

input = sys.stdin.readline

n = int(input())
answer = 0

for a in range(1, n // 3 + 1):
    lower = max(a, n // 2 - a + 1)
    upper = (n - a) // 2

    if lower <= upper:
        answer += upper - lower + 1

print(answer)
```

### 설명

세 변을 `a <= b <= c`라고 두고 `a + b + c = N`이라고 하면, `c = N - a - b`이다.

여기서 조건을 정리하면

- `b <= c` 이므로 `b <= (N - a) // 2`
- `a + b > c` 이므로 `b > N/2 - a`
- 또 `a <= b`도 만족해야 한다.

그래서 `b`는 아래 범위 안에 들어가야 한다.

- 시작값: `max(a, N // 2 - a + 1)`
- 끝값: `(N - a) // 2`

즉, `a` 하나를 정할 때마다 가능한 `b`의 개수를 **범위 길이로 바로 더할 수 있다.**

### 이 풀이가 좋은 이유

- 모든 `(a, b)`를 직접 확인하지 않는다.
- 각 `a`마다 상수 시간 계산만 한다.
- 브루트포스보다 훨씬 빠르고, 제한에도 안정적이다.

## 개선 풀이 2: 수학 공식으로 더 줄이기

이 문제는 더 나아가면 **정수 삼각형 개수 공식**으로도 처리할 수 있다.

```python
import sys

input = sys.stdin.readline

n = int(input())

if n % 2 == 0:
    print((n * n + 24) // 48)
else:
    print(((n + 3) * (n + 3) + 24) // 48)
```

### 설명

이 공식은 둘레가 `N`인 정수 삼각형의 개수를 정리한 결과를 그대로 사용한 것이다.

- `N`이 짝수일 때는 `N^2 / 48`에 가장 가까운 정수
- `N`이 홀수일 때는 `(N + 3)^2 / 48`에 가장 가까운 정수

를 구하면 된다.

코드에서 `+24`를 더한 뒤 `// 48`을 하는 이유는  
`48`로 나누는 값을 정수 반올림처럼 처리하기 위해서다.

이 방식은 가장 빠르지만,  
문제를 처음 이해하는 단계에서는 `개선 풀이 1`처럼 범위를 직접 세는 방식이 더 직관적이라고 느껴졌다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(N^2)`
- 개선 풀이 1: `O(N)`
- 개선 풀이 2: `O(1)`

왜 이렇게 되는지도 같이 보면 더 명확하다.

- 처음 풀이는 바깥 루프와 안쪽 루프를 통해 변 조합을 직접 확인하므로 전체적으로 이차 시간에 가깝다.
- 개선 풀이 1은 `a`를 한 번만 순회하고, 각 단계에서 가능한 `b`의 개수를 상수 시간에 계산하므로 `O(N)`이다.
- 개선 풀이 2는 입력 하나에 대해 사칙연산 몇 번만 수행하므로 입력 크기가 커져도 연산 수가 늘지 않아 `O(1)`이다.

## 이번 문제에서 배운 점

- 처음에는 직접 세어보는 브루트포스가 가장 자연스럽지만, 조건을 정리하면 훨씬 빠르게 바꿀 수 있다.
- `a <= b <= c`처럼 **중복 제거용 정렬 조건**을 먼저 잡으면 문제를 수학적으로 다루기 쉬워진다.
- 시간 복잡도를 줄이려면 무조건 새로운 자료구조를 쓰는 것이 아니라, **세는 방식을 바꾸는 것**만으로도 충분할 때가 많다.

이번 문제는  
브루트포스에서 시작해서 `O(N)`, 더 나아가 `O(1)`까지 줄일 수 있다는 점이 특히 인상적이었다.

## 마무리

처음 작성한 코드는 “삼각형 조건을 직접 확인한다”는 점에서 문제를 이해하는 데는 도움이 되는 코드였다.  
하지만 제한까지 생각하면, 이 문제는 결국 조건을 식으로 정리해서 더 빠르게 세는 쪽이 훨씬 적절했다.

같은 문제를 다시 정리해보면서

- 직접 세는 코드가 무엇을 하고 있는지
- 왜 느린지
- 같은 조건을 더 짧고 빠르게 셀 수 있는지

를 같이 정리해두면 다음에 비슷한 문제를 만났을 때 훨씬 도움이 될 것 같다.
