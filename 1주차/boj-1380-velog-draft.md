알고팜 1주차에서 풀었던 **BOJ 1380 - 귀걸이**를 정리해보려고 한다.
이번 문제는 로직 자체는 어렵지 않았지만, 여러 시나리오를 처리하는 입력 구조와 출력 형식 때문에 은근히 실수하기 쉬웠다.
이번 글에서는 내가 처음 작성한 코드가 어떤 생각으로 만들어졌는지, 어디가 아쉬웠는지, 그리고 더 깔끔하게 풀 수 있는 방법까지 같이 남겨보겠다.

문제 링크: https://www.acmicpc.net/problem/1380

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현 / 카운팅

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 여러 시나리오가 주어지고, 각 시나리오마다 귀걸이를 압수당한 여학생 수 `n`이 먼저 주어진다.
- 이어서 `n`명의 이름이 순서대로 주어진다. 이 순서가 곧 학생 번호 `1 ~ n`에 대응된다.
- 그다음 `2n - 1`줄 동안 `학생 번호`와 `A` 또는 `B`가 함께 주어진다.
- 정상이라면 한 학생 번호는 두 번 등장해야 하지만, 귀걸이를 끝내 돌려받지 못한 학생은 한 번만 등장한다.
- 각 시나리오마다 등장 횟수가 홀수인 번호를 찾아 `시나리오 번호 학생 이름` 형식으로 출력하면 된다.

핵심은 `A/B` 자체가 아니라, **어떤 번호가 몇 번 등장했는지**를 보는 문제라는 점이다.

## 입력 형식

- 각 시나리오의 첫 줄에 귀걸이를 압수당한 여학생 수 `n`이 주어진다.
- 다음 `n`줄에는 여학생 이름이 한 줄에 하나씩 주어진다.
- 다음 `2n - 1`줄에는 `번호 문자` 형식의 기록이 주어진다.
- 마지막에는 `0`이 주어지고, 이 줄은 처리하지 않고 입력을 종료한다.
- 문제 조건상 `1 <= n <= 100`이고, 이름 길이는 최대 60자이다.

## 출력 형식

- 각 시나리오마다 `시나리오 번호`와 `귀걸이를 돌려받지 못한 학생 이름`을 공백 하나로 구분해 출력한다.
- 시나리오 번호는 `1`부터 시작해서 입력 순서대로 증가한다.

## 예제 입력 1

```text
3
Betty Boolean
Alison Addaway
Carrie Carryon
1 B
2 A
3 B
3 A
1 A
2
Helen Clark
Margaret Thatcher
1 B
2 B
2 A
0
```

## 예제 출력 1

```text
1 Alison Addaway
2 Helen Clark
```

## 접근 아이디어

이 문제는 자료구조를 복잡하게 쓸 필요가 없다.

1. 학생 이름을 번호와 대응되도록 저장한다.
2. 기록 `2n - 1`개를 읽으면서 각 번호의 등장 횟수를 센다.
3. 끝까지 읽은 뒤 한 번만 등장한 번호를 찾아 해당 이름을 출력한다.

즉, 한 시나리오마다 결국 필요한 것은

- 번호별 등장 횟수
- 번호에 대응되는 이름

이 두 가지뿐이다.

## 내가 처음 작성한 코드

```python
#1주차 과제 BOJ 1380: 귀걸이
#https://www.acmicpc.net/problem/1380
import sys

input = sys.stdin.readline

case = 1

name_correct = {}
while True:
    n = int(input())
    
    if n == 0:
        break

    names = [""] * (n + 1)
    count = [0] * (n + 1)

    # 이름
    for i in range(1, n + 1):
        names[i] = input()

    # 귀걸이 기록
    for _ in range(2 * n - 1):
        num, _ = input().split()
        count[int(num)] += 1

    # 한 번만 나온 학생 찾기
    for i in range(1, n + 1):
        if count[i] == 1:
            name_correct[case] = names[i]
            break
            

    case += 1

for i in range(1, case):
    print(i, name_correct[i])
```

## 이 코드에서 내가 하려고 했던 것

처음 코드의 방향 자체는 문제 핵심을 잘 잡고 있었다.

- 학생 이름을 `1 ~ n` 인덱스로 저장한다.
- 기록에서 번호만 뽑아 등장 횟수를 센다.
- 마지막에 한 번만 등장한 번호를 찾아 그 이름을 출력한다.

특히 `A`와 `B`는 그대로 무시하고 번호만 세는 방식은 이 문제에 잘 맞는다.
정상적으로 귀걸이를 돌려받은 학생은 같은 번호가 두 번 등장하고, 못 돌려받은 학생만 한 번 등장하기 때문이다.

즉, 큰 아이디어는 맞았고 실제로도 거의 정답에 가까운 코드였다.

## 아쉬웠던 점 또는 반례

가장 큰 문제는 **이름을 입력받을 때 줄바꿈 문자를 그대로 저장했다는 점**이다.

```python
names[i] = input()
```

`sys.stdin.readline()`을 쓰고 있으므로 `names[i]`에는 이름 뒤에 `\n`이 포함된다.
그 상태에서 마지막에 아래처럼 출력하면,

```python
print(i, name_correct[i])
```

이름 안에 들어 있던 줄바꿈과 `print()`의 줄바꿈이 겹쳐서 **시나리오 사이에 빈 줄이 하나씩 더 생긴다.**

공식 예제를 그대로 넣으면 출력이 이런 모양이 된다.

```text
1 Alison Addaway

2 Helen Clark
```

정답은 빈 줄 없이 두 줄이어야 하므로, 이 코드는 출력 형식 때문에 오답이 날 수 있다.

그 외에도 조금 아쉬운 점이 있다.

- `name_correct` 딕셔너리에 모든 정답을 저장한 뒤 마지막에 다시 출력할 필요는 없다.
- 한 시나리오가 끝날 때 바로 출력하면 코드가 더 단순해진다.
- `count[i] == 1`도 맞지만, 이 문제의 핵심이 "홀수 번 등장"이라는 점을 드러내려면 `count[i] % 2 == 1`처럼 써도 의미가 더 잘 보인다.

## 개선 풀이 1

가장 직관적인 개선은 기존 아이디어를 그대로 유지하되, 입력과 출력을 안전하게 다듬는 것이다.

```python
import sys

input = sys.stdin.readline

case = 1

while True:
    n = int(input())
    if n == 0:
        break

    names = [""] + [input().rstrip() for _ in range(n)]
    count = [0] * (n + 1)

    for _ in range(2 * n - 1):
        num, _ = input().split()
        count[int(num)] += 1

    for i in range(1, n + 1):
        if count[i] % 2 == 1:
            print(case, names[i])
            break

    case += 1
```

### 설명

- 이름을 받을 때 `rstrip()`으로 줄바꿈을 제거했다.
- 딕셔너리에 정답을 모아둘 필요 없이, 시나리오 하나가 끝날 때 바로 출력했다.
- 핵심 로직은 원래 코드와 같고, 구현만 더 안전하게 정리한 버전이다.

원래 코드의 장점을 거의 그대로 가져가면서, 실제 제출에서 문제 될 수 있는 출력 이슈를 없앤 풀이라고 볼 수 있다.

## 개선 풀이 2

이 문제는 등장 횟수를 모두 저장하지 않고도 풀 수 있다.
같은 번호가 두 번 나오면 상쇄된다는 점을 이용해서 XOR로 처리할 수 있다.

```python
import sys

input = sys.stdin.readline

case = 1

while True:
    n = int(input())
    if n == 0:
        break

    names = [""] + [input().rstrip() for _ in range(n)]
    missing = 0

    for _ in range(2 * n - 1):
        num, _ = input().split()
        missing ^= int(num)

    print(case, names[missing])
    case += 1
```

### 설명

- 같은 번호가 두 번 나오면 `x ^ x = 0`이 되어 사라진다.
- 결국 한 번만 등장한 번호만 XOR 결과에 남는다.
- `A/B`는 여전히 중요하지 않고, 번호만 읽으면 된다.

개인적으로는 가독성 면에서는 첫 번째 개선 풀이가 더 무난하고,
문제 성질을 한 번 더 활용해보고 싶다면 두 번째 풀이도 꽤 깔끔하다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(n)`
- 개선 풀이 1: `O(n)`
- 개선 풀이 2: `O(n)`

이유는 세 풀이 모두 한 시나리오에서

- 이름 `n`개를 한 번 읽고
- 기록 `2n - 1`개를 한 번 처리하고
- 필요하면 번호 배열을 한 번 훑는

구조이기 때문이다.

즉, 중첩 반복으로 `n^2`이 되는 부분은 없고, 입력 크기에 비례해서 한두 번 순회하는 정도라서 전체 시간 복잡도는 `O(n)`이다.

공간 복잡도는 조금 차이가 있다.

- 처음 풀이: 이름 배열과 카운트 배열을 써서 `O(n)`
- 개선 풀이 1: 이름 배열과 카운트 배열을 써서 `O(n)`
- 개선 풀이 2: 이름 배열은 필요하지만 카운트 배열 없이 XOR 변수 하나만 더 써서 추가 작업 메모리는 `O(1)`

## 이번 문제에서 배운 점

- 문제에서 진짜 중요한 정보가 무엇인지 먼저 분리해야 한다. 이 문제에서는 `A/B`보다 `번호의 등장 횟수`가 핵심이었다.
- 로직이 맞아도 입력과 출력 처리에서 실수하면 바로 오답이 된다. 특히 `readline()`을 쓸 때 줄바꿈 처리를 습관처럼 확인해야겠다.
- 처음 풀이를 완전히 버리기보다, 어떤 아이디어는 맞았고 어떤 부분만 고치면 되는지 나눠서 보는 연습이 도움이 된다.
