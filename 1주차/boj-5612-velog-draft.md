알고팜 1주차에서 풀었던 문제 중 하나인 **BOJ 5612 - 터널의 입구와 출구**를 정리해보려고 한다.  
이번 글에서는 문제를 다시 짚어보고, 내가 처음 작성한 코드가 어떤 흐름으로 동작하는지 설명한 뒤, **더 정확하고 단순하게 풀 수 있는 개선 방식**도 함께 남겨보겠다.

문제 링크: https://www.acmicpc.net/problem/5612

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현 / 시뮬레이션

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 처음에 터널 안에는 차량이 `m`대 있다.
- `n`분 동안, 매 1분마다 `입구로 들어온 차량 수`, `출구로 나온 차량 수`가 주어진다.
- 매 분이 지날 때마다 터널 안 차량 수를 갱신한다.
- 조사 시작 시점부터 조사 종료 시점까지, 터널 안 차량 수의 **최댓값**을 구해야 한다.
- 단, 계산 중 한 번이라도 차량 수가 음수가 되면 관측이 잘못된 것이므로 `0`을 출력한다.

핵심은 결국 두 가지다.

- 현재 차량 수를 정확하게 추적하기
- 음수가 된 적이 있는지와 최댓값을 분리해서 관리하기

## 입력 형식

- 첫째 줄에 조사한 시간 `n`이 주어진다.
- 둘째 줄에 조사를 시작할 때 터널 안에 있던 차량 수 `m`이 주어진다.
- 다음 `n`개의 줄에는 각 1분 동안의 `입구 통과 차량 수`, `출구 통과 차량 수`가 주어진다.
- `n <= 10000`, 1분 동안 입구/출구를 통과하는 차량 수는 각각 `100` 이하이다.

## 출력 형식

- 조사 시작 후 `j`분이 지난 시점의 터널 안 차량 수를 `S_j`라고 할 때, `S_j`의 최댓값을 출력한다.
- 단, 어느 시점이든 차량 수가 음수가 되면 `0`을 출력한다.

## 예제 입력 1

```text
3
2
2 3
2 3
4 1
```

## 예제 출력 1

```text
3
```

## 예제 입력 2

```text
3
2
2 3
2 4
4 1
```

## 예제 출력 2

```text
0
```

## 예제 입력 3

```text
3
2

```

## 예제 출력 3

```text
2
```

## 접근 아이디어

이 문제는 복잡한 자료구조가 필요한 문제가 아니다.

1. 현재 차량 수를 계속 갱신한다.
2. 갱신한 값이 음수가 되는지 확인한다.
3. 음수가 아니라면 최댓값을 갱신한다.

즉, 매 분마다 상태 하나만 잘 추적하면 되는 전형적인 시뮬레이션 문제라고 볼 수 있다.

## 내가 처음 작성한 코드

```python
#5612번: 터널의 입구와 출구
#https://www.acmicpc.net/problem/5612

import sys

input = sys.stdin.readline

n = int(input()) #조사한 시간
m = int(input()) #현재 터널 안의 차량 수
maximum = m #최대 차량 수
sj = []

for i in range(n):
    a, b = list(map(int, input().split())) #입구와 출구를 통과한 차량 수
    m += a - b
    if m < 0:
        maximum = 0
    if m > maximum:
        maximum = m
    sj.append(maximum)

dump = []
for j in sj:
    if j == 0:
        dump.append(0)
    else:
        dump.append(max(sj))

if 0 in dump:
    print(0)
else:
    print(max(dump))
```

## 이 코드에서 내가 하려고 했던 것

처음에는 아래 흐름으로 생각했다.

1. 현재 차량 수 `m`에 대해 매 시간 `들어온 차량 - 나간 차량`을 반영한다.
2. 매 시점마다 최댓값 `maximum`을 갱신한다.
3. 그 최댓값을 `sj` 리스트에 저장한다.
4. 이후 `dump` 리스트를 만들어서, 중간에 이상한 상황이 있었는지 확인하려고 했다.

아이디어 자체는 나쁘지 않았다.  
특히 `현재 차량 수를 갱신하면서 최댓값을 같이 추적한다`는 방향은 이 문제의 핵심과 맞닿아 있다.

## 아쉬웠던 점

문제는 `0이 등장했는가?`를 기준으로 비정상 상황을 판별한 부분이다.

이 문제에서 정말 확인해야 하는 것은 아래 둘이다.

- **현재 차량 수가 음수가 되었는가**
- **음수가 아니라면 최댓값이 얼마인가**

그런데 내 코드에서는 `maximum == 0`인 상황과 `현재 차량 수가 음수가 된 상황`을 사실상 비슷하게 취급하고 있다.  
이 둘은 완전히 다른 상황이다.

예를 들어 아래 입력을 보자.

```text
2
0
0 0
5 4
```

이 경우 실제 흐름은 다음과 같다.

- 시작 차량 수: 0
- 1시간 뒤: 0
- 2시간 뒤: 1
- 정답: 1

하지만 처음 코드처럼 `0이 한 번이라도 등장했는지`를 별도로 검사하면, 음수가 된 적이 없어도 `0`을 출력할 수 있다.

즉, 이 문제는

- `최댓값의 변화 기록`을 여러 리스트로 남길 필요가 없고
- 매 순간의 **현재 차량 수**만 정확히 추적하면 된다.

## 개선 풀이 1: 가장 직관적인 방법

가장 깔끔한 방식은 현재 차량 수와 최댓값만 관리하는 것이다.

```python
import sys

input = sys.stdin.readline

n = int(input())
current = int(input())
answer = current

for _ in range(n):
    enter, leave = map(int, input().split())
    current += enter - leave

    if current < 0:
        print(0)
        break

    answer = max(answer, current)
else:
    print(answer)
```

### 설명

- `current`는 현재 터널 안 차량 수다.
- 매 입력마다 `enter - leave`를 반영한다.
- 만약 `current < 0`이면 바로 `0`을 출력하고 종료한다.
- 아니라면 `answer = max(answer, current)`로 최댓값만 갱신하면 끝이다.

### 이 풀이가 좋은 이유

- 리스트가 필요 없다.
- 문제 조건을 그대로 코드에 옮긴 형태라 읽기 쉽다.
- 시간 복잡도는 `O(n)`, 추가 공간은 `O(1)`이다.

## 개선 풀이 2: 리스트로 관리하고 싶다면

내가 처음 생각했던 것처럼 리스트를 쓰고 싶다면, `최댓값 리스트`가 아니라 **매 시점의 실제 차량 수**를 저장하는 편이 더 자연스럽다.

```python
import sys

input = sys.stdin.readline

n = int(input())
cars = [int(input())]

for _ in range(n):
    enter, leave = map(int, input().split())
    cars.append(cars[-1] + enter - leave)

if min(cars) < 0:
    print(0)
else:
    print(max(cars))
```

### 이 방식의 포인트

- `cars`에는 각 시점의 실제 차량 수가 들어간다.
- 따라서 `min(cars) < 0`이면 비정상 상황을 바로 확인할 수 있다.
- `max(cars)`는 자연스럽게 조사 시간 동안의 최댓값이 된다.

다만 이 문제는 매 순간 값만 보면 되기 때문에, 개인적으로는 첫 번째 개선 풀이가 더 적절하다고 느꼈다.

## 이번 문제에서 배운 점

- 문제 조건에서 검사해야 하는 값이 **무엇인지 정확히 잡는 것**이 중요하다.
- `최댓값`, `중간 상태`, `실패 조건`은 서로 다른 개념인데, 이를 섞어서 처리하면 반례가 생기기 쉽다.
- 리스트를 많이 쓰는 것보다, 정말 필요한 상태만 남기는 쪽이 오히려 더 정확하고 단순할 수 있다.

이번 문제는 난도가 높지는 않았지만,  
오히려 그래서 더 기본적인 포인트를 점검하기 좋았다.

- 현재 상태를 정확히 추적하기
- 실패 조건을 별도로 분명하게 처리하기
- 불필요한 자료구조를 줄이기

앞으로도 알고팜 문제를 풀 때는  
내가 지금 추적해야 하는 값이 정확히 무엇인지부터 먼저 정리하고 들어가야겠다.
