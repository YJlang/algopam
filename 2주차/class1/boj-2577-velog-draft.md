# [알고팜 2주차] BOJ 2577 숫자의 개수 - 맞는 풀이였지만 `count()`를 너무 많이 불렀다

이번 문제는 처음 풀 때 Python 문자열 메서드가 편해서 금방 통과했는데, 다시 보니 같은 문자열을 여러 번 다시 세는 구조였다. 정답은 맞았지만, 한 번만 순회해서 숫자 개수를 세는 방식이 더 깔끔해서 같이 정리해두고 싶었다.

문제 링크: https://www.acmicpc.net/problem/2577

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현, 문자열, 자릿수 세기

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 자연수 `A`, `B`, `C`가 각각 한 줄씩 주어진다.
- 세 수를 곱한 결과를 만든다.
- 그 결과 안에 숫자 `0`부터 `9`가 각각 몇 번 등장하는지 세어야 한다.
- 출력은 `0`의 개수부터 `9`의 개수까지 순서대로 한 줄씩 하면 된다.

## 입력 형식

- 첫째 줄에 `A`, 둘째 줄에 `B`, 셋째 줄에 `C`가 주어진다.
- `A`, `B`, `C`는 모두 100 이상 1000 미만의 자연수이다.

## 출력 형식

- 첫째 줄부터 열 번째 줄까지 `0`부터 `9`까지 각 숫자의 등장 횟수를 차례대로 출력한다.

## 예제 입력 1

```text
150
266
427
```

## 예제 출력 1

```text
3
1
0
2
0
0
0
2
0
0
```

## 접근 아이디어

1. 먼저 세 수를 곱해서 결과 숫자를 만든다.
2. 그 결과를 문자열로 바꾸면 각 자리 숫자를 문자처럼 다루기 쉬워진다.
3. 각 숫자가 몇 번 나오는지 세는 방법은 여러 가지가 있는데, 내 처음 풀이는 `count()`를 반복해서 쓰는 방식이었다.

## 내가 처음 작성한 코드

```python
#BOJ 2577: 숫자의 개수
#https://www.acmicpc.net/problem/2577
#브론즈2
import sys

input = sys.stdin.readline

a = int(input()) #시간복잡도 O(1)
b = int(input()) #시간복잡도 O(1)
c = int(input()) #시간복잡도 O(1)

result = a * b * c #시간복잡도 O(1)
result = str(result) #시간복잡도 O(n)

for i in range(10): #시간복잡도 O(10)
    print(result.count(str(i))) #count는 시간복잡도 O(n)
```

## 이 코드에서 내가 하려고 했던 것

세 수를 곱한 값을 문자열로 바꾸고, `0`부터 `9`까지 하나씩 `count()`로 세어서 출력하려는 코드였다. 문제를 푸는 방향 자체는 정확했고, 예제 입력 `150, 266, 427`을 넣어도 정답 출력이 잘 나온다.

문자열로 바꾼 뒤 자릿수를 세겠다는 발상도 괜찮았다. 숫자를 직접 나누면서 자릿수를 꺼내지 않아도 되고, Python에서는 문자열 메서드가 잘 되어 있어서 구현이 짧아진다.

## 아쉬웠던 점

가장 아쉬운 부분은 `result.count(str(i))`를 10번 반복한다는 점이다. `count()`는 문자열 처음부터 끝까지 보면서 해당 문자가 몇 개인지 세기 때문에, 숫자 하나를 셀 때마다 문자열 전체를 다시 훑는다.

즉, 같은 문자열을 `0`, `1`, `2`, ..., `9`에 대해 계속 다시 확인하고 있는 셈이다. 결과 문자열 길이가 길어질수록 이런 반복 스캔이 조금씩 더 아깝게 느껴진다.

또 코드에 적어 둔 시간복잡도 주석도 한 줄씩 보면 맞는 편이지만, 전체 흐름으로 보면 `for`문이 고정 10번 돈다고 끝이 아니라, 그 안에서 매번 `O(n)`짜리 `count()`가 실행된다. 그래서 전체적으로는 `O(10n)`이고 Big-O로는 `O(n)`이다.

```text
반례가 있어서 틀리는 코드는 아니었다.
약점은 정답 여부보다도, 같은 문자열을 여러 번 다시 세는 비효율에 가까웠다.
```

## 개선 풀이 1

```python
import sys
input = sys.stdin.readline

a = int(input())
b = int(input())
c = int(input())

product = str(a * b * c)
counts = [0] * 10

for ch in product:
    # 문자 숫자를 정수로 바꿔서 해당 칸의 개수를 1 늘린다.
    counts[int(ch)] += 1

for cnt in counts:
    print(cnt)
```

이 방법은 곱한 결과 문자열을 한 번만 순회하면서 각 숫자의 등장 횟수를 바로 배열에 누적한다. `counts[3]`은 숫자 3이 나온 횟수, `counts[7]`은 숫자 7이 나온 횟수처럼 바로 의미가 대응돼서 읽기도 편하다.

`count()`를 10번 호출하는 대신, 문자열을 한 번만 본다는 점이 가장 큰 차이다.

## 개선 풀이 2

```python
import sys
input = sys.stdin.readline

a = int(input())
b = int(input())
c = int(input())

num = a * b * c
counts = [0] * 10

while num > 0:
    # 맨 뒤 자릿수를 꺼내서 개수를 세고,
    counts[num % 10] += 1
    # 마지막 자릿수는 제거한다.
    num //= 10

for cnt in counts:
    print(cnt)
```

이 버전은 문자열로 바꾸지 않고, 숫자 자체를 10으로 나눈 나머지와 몫을 이용해서 자릿수를 하나씩 꺼내는 방식이다. `num % 10`은 맨 뒤 자릿수이고, `num //= 10`은 그 자릿수를 제거하는 역할을 한다.

이 문제에서는 `A`, `B`, `C`가 모두 100 이상이어서 곱한 결과가 0이 되는 경우가 없으니 이렇게 바로 써도 안전하다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(n)`
- 개선 풀이 1: `O(n)`
- 개선 풀이 2: `O(n)`

여기서 `n`은 `A * B * C`를 문자열이나 자릿수 기준으로 봤을 때의 전체 자리 수이다.

- 내가 처음 작성한 풀이는 `count()`가 한 번 호출될 때마다 결과 전체를 끝까지 확인한다. 이 작업을 숫자 0부터 9까지 총 10번 반복하므로 `O(10n)`이고, Big-O로는 `O(n)`이다.
- 개선 풀이 1은 결과 문자열을 처음부터 끝까지 한 번만 보면서 개수를 누적하므로 `O(n)`이다.
- 개선 풀이 2도 각 자릿수를 한 번씩만 꺼내서 세므로 `O(n)`이다.

조금 더 쉽게 말하면, 원래 풀이는 결과 자릿수가 2배가 되면 한 번 세는 데 걸리는 시간도 2배쯤 늘어나고, 그걸 10번 반복한다. 반면 개선 풀이들은 결과를 한 번만 훑기 때문에 같은 `O(n)`이어도 실제 반복 작업의 양이 더 적다.

참고로 이 문제의 입력 범위에서는 `A`, `B`, `C`가 모두 세 자리 이하라 곱한 결과의 자릿수도 크지 않다. 그래서 실제 채점에서는 세 방법 모두 충분히 빠르다. 그래도 어떤 반복이 중복되는지는 짚고 넘어가면 다음 문제에서 도움이 된다.

## 이번 문제에서 배운 점

- Python의 `count()`는 편하지만, 반복해서 부르면 같은 문자열을 여러 번 다시 확인하게 된다.
- 같은 `O(n)`이라도 한 번 순회하느냐 여러 번 순회하느냐에 따라 실제 작업량은 달라질 수 있다.
- 숫자 세기 문제는 문자열로 처리할 수도 있고, `% 10`과 `// 10`으로 자릿수를 직접 꺼내서 처리할 수도 있다.
