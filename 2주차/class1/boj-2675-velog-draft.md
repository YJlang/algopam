# [알고팜 2주차] BOJ 2675 문자열 반복 - 정답은 맞지만 `+=` 누적은 조금 아쉬웠다

이번 문제는 구현 자체는 어렵지 않았지만, 문자열을 다루는 습관을 점검하기에 꽤 좋은 문제였다.  
내 원래 코드도 정답은 맞았지만, `result += ...` 방식으로 문자열을 계속 이어 붙인 점은 나중에 더 큰 입력을 만났을 때 아쉬운 습관이 될 수 있어서 같이 정리해봤다.

문제 링크: https://www.acmicpc.net/problem/2675

## 문제 설명

문자열 `S`와 반복 횟수 `R`이 주어지면, `S`의 각 문자를 `R`번씩 반복해서 새 문자열 `P`를 만들어 출력하는 문제다.

- 첫 번째 문자도 `R`번 반복
- 두 번째 문자도 `R`번 반복
- 이런 식으로 끝까지 반복해서 새 문자열을 만든다
- 테스트 케이스가 여러 개 주어질 수 있다

## 입력 형식

- 첫째 줄에 테스트 케이스 개수 `T`가 주어진다
- 각 테스트 케이스는 `R`과 문자열 `S`가 공백으로 구분되어 주어진다
- `R`은 1 이상 8 이하이고, `S`의 길이는 1 이상 20 이하이다

## 출력 형식

- 각 테스트 케이스마다 새로 만든 문자열 `P`를 한 줄씩 출력한다

## 예제 입력 1

```text
2
3 ABC
5 /HTP
```

## 예제 출력 1

```text
AAABBBCCC
/////HHHHHTTTTTPPPPP
```

## 접근 아이디어

1. 테스트 케이스 수 `T`만큼 반복한다.
2. 각 테스트 케이스에서 반복 횟수 `R`과 문자열 `S`를 입력받는다.
3. 문자열 `S`를 한 글자씩 보면서 각 문자를 `R`번 반복해 새 문자열에 붙인다.

이 문제에서 핵심은 "문자열 전체를 반복하는 것"이 아니라 "문자 하나하나를 반복해야 한다"는 점이다.  
예를 들어 `ABC`를 3번 반복하면 `ABCABCABC`가 아니라 `AAABBBCCC`가 되어야 한다.

## 내가 처음 작성한 코드

아래 코드는 내가 제출한 로직을 블로그용으로 주석만 조금 정리한 버전이다.

```python
import sys

input = sys.stdin.readline

t = int(input())  # 테스트 케이스 개수

for _ in range(t):
    r, s = input().split()  # r: 반복 횟수, s: 문자열
    r = int(r)

    result = ""
    for char in s:
        result += char * r  # 문자 하나를 r번 반복해서 누적

    print(result)
```

## 이 코드에서 내가 하려던 것

내 코드는 문제 의도 자체는 정확히 따라가고 있다.

- 문자열 `S`를 한 글자씩 꺼낸다
- 각 글자에 `r`을 곱해서 반복 문자열을 만든다
- 그 결과를 `result`에 차례대로 이어 붙인다
- 테스트 케이스마다 완성된 문자열을 출력한다

즉, "문자 단위 반복"을 제대로 구현한 정답 코드다.

## 아쉬웠던 점

이 코드는 이 문제에서는 충분히 통과한다.  
입력 제한이 작아서 실제로 시간 초과가 날 일도 거의 없다.

그래도 습관 측면에서는 조금 아쉬운 부분이 있다.

```python
result += char * r
```

파이썬 문자열은 한 번 만들어지면 내부 내용을 직접 바꿀 수 없는 자료형이다.  
그래서 `result += ...`를 반복하면, 매번 "기존 문자열 + 새 문자열"을 다시 만든다고 생각하는 편이 안전하다.

문자열이 짧을 때는 체감이 안 나지만, 길어질수록 이미 만든 앞부분을 계속 복사하는 일이 생긴다.  
이 문제는 정답은 맞지만, 더 큰 문자열 문제에서는 이런 방식이 비효율적으로 바뀔 수 있다.

즉, 이 문제는 "틀린 풀이"라기보다 "맞는 풀이지만 더 좋은 습관이 있는 풀이"라고 보는 게 맞았다.

## 개선 코드 1

가장 무난한 방법은 반복된 조각을 리스트에 모아두고, 마지막에 `''.join()`으로 한 번에 합치는 것이다.

```python
import sys

input = sys.stdin.readline

t = int(input())

for _ in range(t):
    r, s = input().split()
    r = int(r)

    pieces = []  # 반복된 조각들을 담아둘 리스트
    for char in s:
        pieces.append(char * r)  # 각 문자를 r번 반복한 뒤 저장

    print("".join(pieces))  # 리스트의 문자열들을 한 번에 이어 붙임
```

여기서 `''.join(pieces)`는 리스트 안에 들어 있는 문자열 조각들을 하나의 문자열로 합쳐 준다.  
중간 결과를 계속 새로 만드는 방식보다 의도가 더 분명하고, 문자열 누적에서도 더 안전한 편이다.

## 개선 코드 2

조금 더 짧게 쓰면 `join` 안에서 바로 반복 문자열을 만들 수도 있다.

```python
import sys

input = sys.stdin.readline

t = int(input())

for _ in range(t):
    r, s = input().split()
    r = int(r)

    # s를 한 글자씩 보면서 char * r 결과를 바로 이어 붙인다
    result = "".join(char * r for char in s)
    print(result)
```

이 방식은 코드가 짧고, "문자별 반복 결과를 이어 붙인다"는 흐름도 잘 보인다.  
개인적으로는 실전에서는 이 버전이 가장 깔끔하다고 느꼈다.

## 시간 복잡도

- 처음 작성한 코드: 테스트 케이스 하나 기준 `O(rn^2)`처럼 볼 수 있다
- 개선 코드 1: 테스트 케이스 하나 기준 `O(rn)`
- 개선 코드 2: 테스트 케이스 하나 기준 `O(rn)`

여기서

- `n`은 문자열 `S`의 길이
- `r`은 각 문자를 반복하는 횟수

를 뜻한다.

### 왜 처음 코드는 더 아쉽다고 볼 수 있을까?

문자 `n`개를 돌면서 매번 `char * r`을 만든다.  
이 자체만 보면 반복 문자열 길이만큼 일이 생기므로 `O(rn)`처럼 보인다.

하지만 문제는 `result += ...` 부분이다.  
문자열을 계속 이어 붙이면, 이미 만들어 둔 앞부분까지 다시 복사하는 비용이 누적될 수 있다.

예를 들어 첫 번째 글자를 붙일 때보다 열 번째 글자를 붙일 때가 더 비싸다.  
앞에서 만들어 둔 결과가 점점 길어지기 때문이다.  
그래서 전체적으로 보면 한 번씩만 처리하는 느낌이 아니라, 뒤로 갈수록 더 큰 문자열을 반복해서 복사하는 흐름이 된다.

입력 문자열 길이가 2배쯤 커지면, 이 누적 복사 비용 때문에 전체 작업량은 2배보다 더 빠르게 늘 수 있다.  
그래서 이 코드는 작은 입력에서는 괜찮지만, 습관적으로는 조심하는 편이 좋다.

### 왜 `join()` 방식은 `O(rn)`일까?

`char * r`로 반복 문자열을 만드는 데 필요한 총 작업량은 결국 출력 문자열 길이와 비슷하다.  
출력 문자열 전체 길이는 `n * r`이므로, 각 문자를 실제로 만들어 내는 데 드는 일은 대략 `O(rn)`이다.

그 다음 `join()`은 이렇게 만들어 둔 조각들을 한 번에 이어 붙인다.  
이미 있는 조각들을 순서대로 읽어서 최종 문자열을 만들기 때문에, 출력 길이에 비례해서 움직인다고 생각하면 된다.

즉, 입력 문자열 길이가 2배가 되면 출력 길이도 대체로 2배가 되고, 필요한 일도 대체로 2배가 된다.  
이런 점에서 `join()` 쪽이 더 예측 가능하고 깔끔하다.

## 이번 문제에서 배운 점

- 이 문제의 핵심은 문자열 전체 반복이 아니라 문자 단위 반복이라는 점이다.
- 파이썬에서 문자열을 반복해서 `+=`로 누적하는 습관은 작은 문제에서는 괜찮아 보여도 나중에 비효율로 이어질 수 있다.
- `''.join()`은 "문자열 조각을 마지막에 한 번에 합친다"는 점에서 문자열 처리 문제에서 자주 쓰이는 안전한 패턴이다.
