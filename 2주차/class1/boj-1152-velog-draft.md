# [알고팜 2주차] BOJ 1152 단어의 개수 - `split()`이 왜 정확하게 먹히는지 정리

이번 문제는 처음 보면 단어를 하나씩 세야 할 것 같지만, Python에서는 `split()`만 제대로 이해하면 아주 짧게 풀 수 있는 문제였다. 다만 "왜 이게 맞는지"를 설명하지 않으면 나중에 다시 봤을 때 그냥 외운 풀이처럼 느껴질 수 있어서, 이번에는 `split()`의 동작과 직접 세는 방법까지 같이 정리해봤다.

문제 링크: https://www.acmicpc.net/problem/1152

## 문제 정보

- 시간 제한: 2초
- 메모리 제한: 128MB
- 풀이 감각: 문자열, 구현

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 영어 대소문자와 공백으로 이루어진 문자열 한 줄이 주어진다.
- 이 문자열 안에 들어 있는 단어의 개수를 세어야 한다.
- 같은 단어가 여러 번 나오면 등장한 횟수만큼 각각 세어야 한다.
- 문자열은 공백으로 시작하거나 끝날 수 있다.

## 입력 형식

- 첫 줄에 영어 대소문자와 공백으로 이루어진 문자열이 주어진다.
- 문자열 길이는 최대 1,000,000이다.
- 단어와 단어 사이는 공백 한 개로 구분된다.
- 공백이 연속해서 나오는 경우는 없다. 다만 맨 앞이나 맨 뒤 공백은 있을 수 있다.

## 출력 형식

- 첫째 줄에 단어의 개수를 출력한다.

## 예제 입력 1

```text
The Curious Case of Benjamin Button
```

## 예제 출력 1

```text
6
```

## 예제 입력 2

```text
 The first character is a blank
```

## 예제 출력 2

```text
6
```

## 예제 입력 3

```text
The last character is a blank 
```

## 예제 출력 3

```text
6
```

## 접근 아이디어

1. 문자열을 공백 기준으로 잘라서 단어 목록을 만든다.
2. 그 목록의 길이를 세면 단어 개수가 된다.
3. Python의 `split()`을 인자 없이 쓰면 앞뒤 공백과 줄바꿈까지 자연스럽게 정리해줘서 이 문제와 잘 맞는다.

## 내가 처음 작성한 코드

```python
#BOJ 1152: 단어의 개수
#https://www.acmicpc.net/problem/1152
#브론즈2
import sys

input = sys.stdin.readline

word = input().split() #split()은 공백을 기준으로 문자열을 분리한다.

print(len(word))
```

## 이 코드에서 내가 하려고 했던 것

문자열을 입력받은 뒤 `split()`으로 단어들을 잘라 리스트로 만들고, 그 리스트 길이를 `len()`으로 세려는 코드였다. 문제 의도와 아주 잘 맞는 풀이이고, 실제로 예제 1, 2, 3 모두 정답이 나온다.

특히 여기서 중요한 점은 `input().split()`을 사용했다는 점이다. `sys.stdin.readline()`으로 입력받으면 문자열 끝에 줄바꿈 문자가 같이 들어오는데, `split()`이 그 부분까지 알아서 정리해 준다.

## 아쉬웠던 점

기능적으로 틀린 부분은 없다. 오히려 이 문제에서는 가장 자연스럽고 파이썬다운 풀이에 가깝다.

다만 주석의 설명을 조금 더 정확하게 쓰면 더 좋겠다고 느꼈다. `split()`을 인자 없이 쓰면 단순히 "공백으로 나눈다" 정도가 아니라, 앞뒤 공백을 무시하고 잘라진 단어들만 남겨 준다. 그래서 입력이 `" The first character is a blank"`처럼 앞에 공백이 있어도 단어 수를 정확히 셀 수 있다.

또 하나는 메모리 관점이다. 이 풀이는 모든 단어를 리스트로 만들어 저장한 뒤 길이를 세기 때문에, 단어 수가 많을수록 그만큼 리스트를 만든다. 이 문제에서는 충분히 통과되지만, 문자열을 한 글자씩 직접 보면서 세면 추가 리스트 없이도 풀 수 있다.

```text
즉, 약점이 있다기보다는 "왜 split()이 이 문제에서 정확한가?"를 더 분명히 이해하면 좋은 코드였다.
```

## 개선 풀이 1

```python
import sys
input = sys.stdin.readline

line = input()

# split()을 인자 없이 쓰면 앞뒤 공백과 줄바꿈을 정리하고 단어만 남긴다.
words = line.split()

print(len(words))
```

로직은 원래 풀이와 같지만, `split()`이 왜 안전한지 조금 더 드러나게 다시 적어본 버전이다. 이 문제를 처음 푸는 입장에서는 "앞뒤 공백이 있어도 왜 괜찮은가?"가 핵심이라서, 그 부분을 주석으로 남겨두는 편이 블로그 글에는 더 친절했다.

## 개선 풀이 2

```python
import sys
input = sys.stdin.readline

line = input().rstrip('\n')
count = 0
in_word = False

for ch in line:
    # 공백이 아닌 문자를 처음 만났다면 새 단어가 시작된 것이다.
    if ch != ' ' and not in_word:
        count += 1
        in_word = True
    elif ch == ' ':
        in_word = False

print(count)
```

이 방법은 문자열을 직접 한 글자씩 보면서 "지금 단어 안에 있는가?"를 추적하는 방식이다. 공백이 아닌 문자를 만났는데 바로 직전까지 단어 밖이었다면, 그 시점이 새 단어의 시작이므로 개수를 1 늘린다.

`split()`처럼 단어 리스트를 따로 만들지 않는다는 점이 차이점이다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(n)`
- 개선 풀이 1: `O(n)`
- 개선 풀이 2: `O(n)`

여기서 `n`은 입력 문자열의 길이이다.

- 내가 처음 작성한 풀이와 개선 풀이 1은 `split()`이 문자열을 처음부터 끝까지 한 번 보면서 단어를 나누기 때문에 `O(n)`이다.
- 개선 풀이 2도 문자열을 한 글자씩 한 번만 확인하므로 `O(n)`이다.

조금 더 쉽게 말하면, 세 풀이 모두 문자열 길이가 2배가 되면 확인해야 하는 문자 수도 대체로 2배가 된다. 반복해서 가장 많은 일을 하는 부분이 결국 문자열 전체를 훑는 과정이라서, 실행 시간은 문자열 길이에 비례해서 늘어난다.

차이가 있다면 개선 풀이 2는 단어 리스트를 만들지 않기 때문에 추가 메모리를 더 아낄 수 있다는 점이다. 시간 복잡도는 같지만, 반복 중에 저장하는 정보의 양은 더 적다.

## 이번 문제에서 배운 점

- Python의 `split()`은 인자 없이 쓰면 앞뒤 공백과 줄바꿈을 정리해 주기 때문에 이런 문제에 특히 강하다.
- 짧게 푸는 것도 좋지만, "왜 맞는지"를 설명할 수 있어야 다음 문자열 문제에서도 응용하기 쉽다.
- 같은 `O(n)` 풀이여도, 리스트를 만드는 방식과 직접 세는 방식은 메모리 사용 측면에서 차이가 날 수 있다.
