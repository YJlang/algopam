# [알고팜 2주차] BOJ 2884 알람 시계 - 조건문 풀이가 왜 맞는지 다시 정리

이번 문제는 처음엔 단순한 시뮬레이션처럼 풀었는데, 다시 보니 "분이 45보다 작은지"만 잘 나누면 꽤 깔끔하게 처리되는 문제였다. 정답은 맞았지만, 같은 문제를 분 단위로 바꿔서 푸는 방법도 같이 정리해두면 이후 시간 계산 문제에서 더 편할 것 같았다.

문제 링크: https://www.acmicpc.net/problem/2884

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현, 시뮬레이션, 시간 계산

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 현재 알람 시각 `H시 M분`이 주어진다.
- 이 시간을 기준으로 45분 더 이른 시각으로 바꿔야 한다.
- 만약 분에서 45를 바로 뺄 수 없으면 한 시간을 빌려와야 한다.
- 시간이 0시보다 작아지면 전날 23시로 돌아간다고 생각하면 된다.

## 입력 형식

- 첫째 줄에 두 정수 `H`, `M`이 공백으로 주어진다.
- `H`는 0 이상 23 이하이다.
- `M`은 0 이상 59 이하이다.

## 출력 형식

- 45분 앞선 시각의 시와 분을 공백으로 출력한다.
- 출력 형식은 입력과 같은 `H M` 형태를 따르면 된다.

## 예제 입력 1

```text
10 10
```

## 예제 출력 1

```text
9 25
```

## 예제 입력 2

```text
0 30
```

## 예제 출력 2

```text
23 45
```

## 예제 입력 3

```text
23 40
```

## 예제 출력 3

```text
22 55
```

## 접근 아이디어

1. `M >= 45`이면 현재 시는 그대로 두고 분만 45를 빼면 된다.
2. `M < 45`이면 한 시간을 빌려와야 하므로 시를 1 줄이고, 분은 `60 - 45 = 15`를 더한 것과 같은 결과로 계산할 수 있다.
3. 시가 `-1`이 되면 하루를 한 바퀴 돈 것이므로 `23`으로 바꿔주면 된다.

## 내가 처음 작성한 코드

```python
#2주차 과제 BOJ2884
#https://www.acmicpc.net/problem/2884

import sys
input = sys.stdin.readline

h,m = map(int, input().split())

if(m < 45):
    m += 15
    h -= 1
else:
    m -= 45

if(h < 0):
    h = 23

print(h, m)
```

## 이 코드에서 내가 하려고 했던 것

핵심 아이디어는 아주 잘 잡혀 있었다. 분이 45보다 작은 경우와 아닌 경우를 나눠서 처리하고, 시간이 음수가 되면 23으로 되돌리는 방식이다.

특히 `m < 45`일 때 `m += 15`로 처리한 부분은 처음 보면 조금 낯설 수 있지만, 사실상 `m = m + 60 - 45`와 같은 계산이다. 즉, 한 시간을 빌려온 뒤 45분을 빼는 흐름을 짧게 쓴 셈이다.

## 아쉬웠던 점

이 코드는 공식 입력 범위 안에서는 기능적으로 틀린 부분이 없다. 샘플 입력 세 개를 넣어봐도 모두 정답이 나온다.

다만 다시 읽을 때 약간 걸리는 부분은 `m += 15`가 왜 맞는지 바로 눈에 들어오지 않을 수 있다는 점이다. 작성할 때는 명확했더라도, 나중에 다시 보면 "왜 15를 더하지?" 하고 한 번 더 생각하게 된다.

또한 이 방식은 "45분 빼기"에는 잘 맞지만, 나중에 90분이나 다른 시간 단위를 빼는 문제로 바뀌면 분기 처리를 다시 설계해야 한다. 그래서 일반화 측면에서는 전체 시간을 분으로 바꾸는 풀이가 더 재사용성이 좋다.

```text
공식 문제 제약 안에서는 반례가 없었다.
아쉬운 점은 정답 여부보다도, 의도가 얼마나 바로 읽히는지와 확장성이었다.
```

## 개선 풀이 1

```python
import sys
input = sys.stdin.readline

h, m = map(int, input().split())
total_minutes = (h * 60 + m - 45) % (24 * 60)

print(total_minutes // 60, total_minutes % 60)
```

현재 시각을 전부 분으로 바꾼 뒤 45를 빼고, 하루 전체 분 수인 `24 * 60`으로 나머지 연산을 해 주는 방식이다. 자정을 넘어가는 경우도 자연스럽게 처리돼서 개인적으로는 가장 일반화된 풀이처럼 느껴졌다.

시간 계산 문제가 조금만 복잡해져도 이 패턴이 훨씬 편해진다.

## 개선 풀이 2

```python
import sys
input = sys.stdin.readline

h, m = map(int, input().split())

if m >= 45:
    print(h, m - 45)
else:
    print((h - 1) % 24, m + 15)
```

원래 풀이의 조건문 구조는 유지하면서, 시간 보정만 `(h - 1) % 24`로 처리한 버전이다. `h`가 0일 때도 자동으로 23이 되기 때문에 후처리용 `if(h < 0)`를 따로 두지 않아도 된다.

조건문으로 푸는 감각은 유지하면서 조금 더 간결하게 정리하고 싶을 때 괜찮았다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(1)`
- 개선 풀이 1: `O(1)`
- 개선 풀이 2: `O(1)`

이 문제는 입력이 고정된 두 정수뿐이고, 세 풀이 모두 반복문 없이 몇 번의 조건 비교와 사칙연산만 수행한다. 즉, 입력 크기가 커진다고 해서 연산 횟수가 늘어나지 않으므로 모두 `O(1)`이다.

## 이번 문제에서 배운 점

- 시간 문제는 `시/분` 상태로 직접 다뤄도 되고, `총 분`으로 바꿔서 더 단순하게 처리할 수도 있다.
- 현재 풀이가 맞더라도, 나중에 다시 읽었을 때 바로 이해되는 표현인지 점검해보는 게 좋다.
- 자정을 넘는 계산은 `% 24`나 `% (24 * 60)`처럼 순환 구조로 처리하면 더 깔끔해질 때가 많다.
