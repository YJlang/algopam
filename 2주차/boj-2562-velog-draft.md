# [알고팜 2주차] BOJ 2562 최댓값 - 맞았지만 다시 보니 아쉬운 첫 풀이

이번 문제는 정답은 금방 맞혔지만, 코드를 다시 보니 불필요한 조건문과 중복 계산이 보여서 한 번 정리해두고 싶었다. 간단한 구현 문제일수록 더 깔끔하게 쓸 수 있는지 돌아보는 연습이 중요하다는 걸 느꼈다.

문제 링크: https://www.acmicpc.net/problem/2562

## 문제 정보

- 시간 제한: 1초
- 메모리 제한: 128MB
- 풀이 감각: 구현, 배열, 최댓값 추적

## 문제 설명

문제 원문은 위 링크에서 확인할 수 있고, 아래는 풀이를 위해 다시 정리한 내용이다.

- 서로 다른 자연수 9개가 한 줄에 하나씩 주어진다.
- 이 중에서 가장 큰 값을 찾아야 한다.
- 최댓값만 출력하는 것이 아니라, 그 값이 입력된 순서도 함께 출력해야 한다.
- 예시에서는 85가 가장 크고, 8번째 수이므로 `85`와 `8`을 출력한다.

## 입력 형식

- 첫째 줄부터 아홉 번째 줄까지 한 줄에 하나의 자연수가 주어진다.
- 주어지는 자연수는 모두 100보다 작다.
- 모든 수는 서로 다르다.

## 출력 형식

- 첫째 줄에 최댓값을 출력한다.
- 둘째 줄에 그 최댓값이 몇 번째 수인지 출력한다.

## 예제 입력 1

```text
3
29
38
12
57
74
40
85
61
```

## 예제 출력 1

```text
85
8
```

## 접근 아이디어

1. 숫자 9개를 저장한 뒤 `max()`로 최댓값을 찾는다.
2. 찾은 최댓값이 리스트의 몇 번째에 있는지 `index()`로 구한다.
3. 다만 이 문제는 한 번 순회하면서 최댓값과 위치를 동시에 갱신해도 풀 수 있어서, 그 방식이 더 단순하고 안전하다.

## 내가 처음 작성한 코드

```python
#2주차 과제 BOJ2562
#https://www.acmicpc.net/problem/2562

import sys
input = sys.stdin.readline

num = []
for i in range(9):
    n = int(input())
    if(i < 100):
        num.append(n)

print(max(num))
print(num.index(max(num)) + 1)
```

## 이 코드에서 내가 하려고 했던 것

기본 방향 자체는 괜찮았다. 입력을 9번 받아 리스트에 넣고, `max(num)`으로 가장 큰 값을 찾은 다음 `num.index(...) + 1`로 몇 번째인지 출력하려고 했다. 실제로 문제의 정답도 정상적으로 나온다.

즉, "모든 값을 저장하고, 최댓값을 찾고, 그 위치를 다시 찾는다"는 흐름으로 접근한 코드였다.

## 아쉬웠던 점

가장 먼저 보였던 부분은 `if(i < 100)` 조건문이다. 문제에서 100보다 작은지를 확인하고 싶었던 것 같은데, 실제로 비교한 값은 입력값 `n`이 아니라 반복 변수 `i`였다. `i`는 0부터 8까지만 돌기 때문에 이 조건은 항상 참이고, 결과적으로 아무 역할도 하지 않는다.

그리고 `max(num)`을 두 번 호출하고 있다는 점도 조금 아쉽다. 데이터가 9개뿐이라 성능 차이는 거의 없지만, 같은 값을 다시 계산하지 않고 변수에 저장해서 재사용하는 편이 더 읽기 쉽다.

문제의 제약 안에서는 정답이지만, "내가 넣은 조건문이 정말 의도한 검사를 하고 있는가?"를 다시 보게 되는 코드였다.

```text
예를 들어 174 같은 값이 들어와도 현재 코드는 i < 100만 검사하므로 그대로 리스트에 추가된다.
물론 이 문제에서는 입력이 100보다 작은 자연수로 보장되기 때문에 실제 채점에는 영향이 없다.
```

## 개선 풀이 1

```python
import sys
input = sys.stdin.readline

max_value = int(input())
max_index = 1

for idx in range(2, 10):
    value = int(input())
    if value > max_value:
        max_value = value
        max_index = idx

print(max_value)
print(max_index)
```

첫 번째 입력값을 기준값으로 잡고, 나머지 8개를 읽으면서 더 큰 값이 나오면 최댓값과 위치를 함께 갱신하는 방식이다. 리스트를 만들 필요가 없어서 흐름이 더 직접적이다.

특히 이 방식은 "입력 하나를 읽을 때마다 현재 최댓값과 비교한다"는 점이 분명해서, 처음 구현 문제를 연습할 때 기준값 갱신 패턴을 익히기 좋다.

## 개선 풀이 2

```python
import sys
input = sys.stdin.readline

numbers = [int(input()) for _ in range(9)]
max_value = max(numbers)

print(max_value)
print(numbers.index(max_value) + 1)
```

리스트를 사용하는 방식이 더 익숙하다면 이렇게 정리할 수도 있다. 원래 코드의 아이디어는 유지하되, 불필요한 조건문을 없애고 `max()`도 한 번만 호출해서 조금 더 깔끔하게 만들었다.

짧게 풀고 싶을 때는 이 방법이 읽기 가장 편했다.

## 시간 복잡도

- 내가 처음 작성한 풀이: `O(n)`
- 개선 풀이 1: `O(n)`
- 개선 풀이 2: `O(n)`

이 문제는 실제로 입력 개수가 항상 9개라서 실행 시간은 사실상 상수처럼 느껴진다. 그래도 입력 개수를 `n`이라고 일반화해서 보면,

- 원래 풀이는 입력을 리스트에 넣는 데 한 번 순회하고, `max(num)`을 두 번, `index()`를 한 번 더 순회하므로 `O(4n)`이고 Big-O로는 `O(n)`이다.
- 개선 풀이 1은 입력을 읽으면서 비교를 한 번씩만 하므로 한 번 순회하는 `O(n)`이다.
- 개선 풀이 2는 리스트를 만드는 데 `O(n)`, `max()`에 `O(n)`, `index()`에 `O(n)`이 들어서 전체적으로 `O(3n)`, 즉 `O(n)`이다.

## 이번 문제에서 배운 점

- 문제 제약과 내가 코드에서 검사하는 조건은 정확히 일치하는지 다시 확인해야 한다.
- 같은 계산을 여러 번 하지 말고, 한 번 구한 값은 변수에 저장해두는 습관이 좋다.
- 간단한 최댓값 문제는 "현재 최댓값과 위치를 함께 갱신한다"는 패턴으로도 자주 풀린다.
